index.js: one calm public doorway into the store layer

After all that machinery, the public module is intentionally almost boring. That is a feature. Most consumers should not care how cloning, path normalization, or proxy caching work. They should just import Store and move on.
  • This file is a barrel with exactly one export, which keeps imports stable even if internals are later reorganized
  • It is the boundary between "private plumbing" and "public contract"
  • That tiny boundary matters in talks and real products alike, because architecture becomes teachable when entry points are obvious
  • In other words: all the complexity lives behind the curtain, and the consumer gets one friendly button to press
 

/**
 * Public exports for the proxy based store layer.
 */
export { Store } from "./_store.js"
