_paths.js: translate between human paths and machine paths

The store lets callers say "filters.search", but the proxy engine wants to work one segment at a time. This file is the translator sitting in the middle, quietly making that API feel friendly.
  • toPathArray() accepts strings, arrays, numbers, symbols, or nothing at all, then normalizes everything into one array shape
  • That means get(), set(), and update() can be flexible at the edges but simple internally
  • pathToString() turns the same structure back into dot notation for event payloads like todos.3.done
  • It is small infrastructure, but it keeps the public API pleasant without making the internals messy
 

export function toPathArray(path) {
  if (Array.isArray(path)) return path
  if (path == null || path === "") return []
  return String(path).split(".").filter(Boolean)
}

export function pathToString(path) {
  return toPathArray(path).join(".")
}
