_guards.js: one tiny question that protects the whole system

This file is almost comically small, and that is exactly why it matters. The store asks this question everywhere: "Is this something I can traverse or proxy, or should I leave it alone?"
  • The guard returns true only for non-null objects, which means primitives stay cheap and untouched
  • The clone layer uses it to stop recursion at strings, numbers, booleans, and null
  • The proxy layer uses it to decide whether a nested value should become another stable proxy
  • This is the kind of boring helper senior engineers love: tiny, reusable, and impossible to misunderstand at 2 AM
 

export function isObject(value) {
  return value !== null && typeof value === "object"
}
