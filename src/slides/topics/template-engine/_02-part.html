_part.js: one tiny base class for reactive DOM slots

A lot of the engine becomes easier to understand once you see this file: a DOM part is just a place that can hold a value, optionally subscribe to a signal, and clean itself up when the source changes.
  • value stores the last committed input, which gives concrete parts a shared place to remember what they rendered
  • signalCleanup stores the unsubscribe function for the currently bound signal, so replacing a binding does not leak subscriptions
  • bindSignal() subscribes once, then immediately pushes signal.get() through the same callback path used for later updates
  • disposeSignal() is boring on purpose: one cleanup path, reused by attribute, property, and child parts
 

export class Part {
  constructor() {
    this.value = undefined
    this.signalCleanup = null
  }

  bindSignal(signal, callback) {
    this.disposeSignal()
    this.signalCleanup = signal.subscribe(() => {
      callback(signal.get())
    })
    callback(signal.get())
  }

  disposeSignal() {
    if (this.signalCleanup) this.signalCleanup()
    this.signalCleanup = null
  }
}
