_event-part.js: the whole event system in one very small file

This slide is a nice reminder that not every runtime primitive needs to be clever. Event bindings are not tracked values. They are simply listener replacement with disciplined cleanup.
  • EventPart does not extend Part, because event listeners are not signal subscriptions and do not share that lifecycle
  • setValue() always removes the old listener first, so rerenders can safely replace closures without stacking handlers
  • Non function values are ignored, which keeps bad bindings harmless instead of half working
  • The class stores exactly one active listener reference, because removal in the DOM API must use the same function identity
 

export class EventPart {
  constructor(element, name) {
    this.element = element
    this.name = name
    this.listener = null
  }

  setValue(value) {
    if (this.listener) {
      this.element.removeEventListener(this.name, this.listener)
      this.listener = null
    }
    if (typeof value !== "function") return
    this.listener = value
    this.element.addEventListener(this.name, this.listener)
  }
}
