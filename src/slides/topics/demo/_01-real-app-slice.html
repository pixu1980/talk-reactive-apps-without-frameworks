components/App/_App.js: one real app shell, not one lonely button

This chapter is where the tiny runtime stops being theory. The app view is a little house made of small rooms, and every room uses the same store, signals, and template helpers we already studied.
  • appView() assembles the header, stats, list, side panels, debug panel, and both modals
  • Every room is just a function that returns html, so composition stays plain and readable
  • No room owns secret state; they all read from the same shared truth box
  • That is why this demo feels like a small product slice, not a stage prop
 

export function appView() {
  return html`
    ${appHeader()}
    

      ${statsRow()}
      ${todoListPanel()}
    

    
${filtersPanel()} ${bulkActionsPanel()}

    
${debugPanel()}

    ${todoModal()} ${categoryModal()}
  `
}
