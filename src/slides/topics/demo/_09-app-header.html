components/AppHeader/_AppHeader.js: big buttons on top, tiny writes underneath

The header looks rich, but its behavior is simple. Buttons call actions, selects use storeModel(), and the rest of the app updates because the same shared store keeps listening.
  • @click=${resetDemo}, @click=${openTodoModal}, and @click=${openCategoryModal} are plain event parts
  • Theme, language, and color scheme selects write straight into preferences.* paths
  • The document dataset effect then mirrors those preferences so CSS and translations can react
  • This proves the runtime is driving the whole shell, not just the list in the middle
