This is the slide where the chapters shake hands. Signals watch, the template engine renders, and the DOM changes for real.
  • The first effect() mirrors theme, color scheme, and language onto the document for CSS and accessibility
  • The second effect() watches tickState and calls render(appView(), root)
  • requestAnimationFrame() measures the app shell after the DOM has been updated
  • So the runtime story is: store change, bell rings, effect runs, template engine paints
