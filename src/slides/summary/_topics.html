What are we going to
build and dissect today?

Topics

We'll cover the following topics:

  • Runtime: Signals, effects, model and repeat
  • Store: Proxy-based global state management
  • Signals: fine-grained reactivity for any value
  • Rendering: DOM-first approach with persistent parts
  • Directives: control flow and custom behavior with tagged templates
  • Demo: building a Todo app from scratch

The runtime core

  • Signals for local dependency graphs, computed values and effect scheduling
  • Proxy Store for path-aware mutations and a stable store:change contract
  • DOM Parts for persistent update slots instead of full tree replacement
  • Tagged Templates for declarative views without JSX or a compiler step

The rendering pipeline

  • model as a tiny two-way binding directive with explicit get/set
  • repeat as keyed range reuse for long lists and stable DOM nodes
  • Generator pipelines for lazy filtering before final materialization
  • Template caching via the stable identity of template literal strings

From primitives to a real demo

  • A Todo app with draft state, filters, category management and bulk actions
  • A live store:change panel wired to the same runtime as the app itself
  • Persistence through localStorage, not through hidden framework state
  • A few subtle bugs that reveal where the runtime still needs engineering discipline
